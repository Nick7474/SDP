# 광명 스마트데이터보드 · UX 개선 작업 로드맵

> **목표:** 현재 디자인을 최대한 유지하면서, 사용자 여정이 물 흐르듯 연결되도록 정리한다.  
> **원칙:** 감사 → 명세 확정(사용자 승인) → 구현 → 검증 순서를 반드시 지킨다.  
> **진행:** 각 단계 완료 시 `[ ]` → `[x]` 로 체크하며 진행 상황을 추적한다.

## Phase 2 · GIS 섹션 심층 감사 ★ 최우선

> GIS 4개 페이지는 필터 → 지도 → 마커 → 팝업 → 드로어 체인이 모두 연결되어야 한다.  
> 다른 페이지와 복잡도 차원이 다르므로 별도 심층 감사를 수행한다.

- [x] **4대 마일 브리핑** — 마일 라디오/탭·공간 레이어 체크박스 → 지도 연동 ✓ / 팝업 버튼 3개 닫기만 ✗
- [x] **탄소·에너지 공간 데이터** — 탭 전환 ✓ / 필터 패널 라디오·체크박스 15개+ 이벤트 없음 ✗ / 핀 팝업 버튼 3개 닫기만 ✗
- [x] **도시 데이터 주제도** — 레이어 트리·표시방식·줌·검색 전부 ✓ / 원천데이터·내려받기 alert ✗ / "관련 지표 보기" 중복 ✗
- [x] **GIS 데이터 관리** — 필터 전체·팝업→드로어 체인·레이어 제어 전부 ✓ (4개 중 완성도 최高) / 미리보기·신규등록·변경이력 alert ✗
- [x] GIS 감사 결과 정리 완료
- [x] **[사용자 검토]** GIS 감사 결과 확인 — 제거 방향 결정

## Phase 3 · UX 명세 작성 (기획)

> 감사 결과를 바탕으로 "이 버튼을 누르면 어떻게 된다"를 문서로 확정한다.  
> 코드를 건드리기 전에 반드시 이 단계를 완료한다.

- [ ] **핵심 사용자 여정 정의** (3~4개)
  ```
  여정 예시:
  ① 운영 점검: 홈 이상 KPI 발견 → 운영 모니터링 → 연계 상태 확인 → 조치
  ② 데이터 조회: GIS 지도 핀 클릭 → 팝업 → 카탈로그 연결 → 데이터셋 확인
  ③ 탄소 보고: 탄소 KPI 확인 → 에너지 공간 데이터 → 내보내기
  ④ GIS 관리: 레이어 등록 → 지도 반영 확인 → 데이터 상태 관리
  ```

- [ ] **페이지별 인터랙션 명세** 작성
  - 각 버튼의 목적지 또는 동작 확정
  - 제거할 요소 목록 확정
  - alert('준비중') 처리 방향 결정 (링크 연결 / 비활성 UI 전환 / 제거)

- [ ] **GIS 필터 ↔ 지도 연동 명세** 작성 (GIS 4페이지 각각)
  ```
  형식 예시:
  [도시 데이터 주제도]
  - 레이어 체크박스 ON → 해당 레이어 마커 지도에 표시
  - 레이어 체크박스 OFF → 해당 레이어 마커 즉시 제거
  - 공간 단위 "동" 선택 → 줌 레벨 7 이동
  - 핀 클릭 → 팝업 오픈 (지도 위 중앙 고정)
  - 팝업 "상세 정보" → 우측 드로어 오픈
  - 팝업 "원천 데이터" → 카탈로그 검색 페이지로 이동
  ```

- [ ] **[사용자 검토 및 승인]** 명세 확정 ← 이 단계 승인 없이 구현 진행 불가

## Phase 4 · 개선 우선순위 결정

> 전체 이슈를 4등급으로 분류하여 구현 순서를 정한다.

- [ ] Must Fix (핵심 동선 — 없으면 사이트가 작동 안 됨)
- [ ] Should Fix (UX 품질 — 있으면 훨씬 나아짐)
- [ ] Nice to Have (추후 개선)
- [ ] Remove (지금 당장 제거할 불필요 요소)

- [ ] **[사용자 검토]** 우선순위 승인

---

## Phase 5 · 구현

> 승인된 명세와 우선순위 기준으로 실제 코드 수정.  
> Must Fix → Should Fix 순서로 진행. 페이지 단위로 완료 후 다음으로 이동.

- [ ] **Must Fix 항목 구현**
  - [ ] 핵심 사용자 여정 4개의 동선 연결 (페이지 간 링크, 버튼 목적지)
  - [ ] GIS 필터 ↔ 지도 연동 수정/보완
  - [ ] 불필요 요소 제거 (Remove 목록)

- [ ] **Should Fix 항목 구현**
  - [ ] alert('준비중') → 비활성 UI 처리 또는 실제 기능 연결
  - [ ] 드로어/모달 "다음 단계" 동선 추가
  - [ ] 페이지 간 컨텍스트 연결 (GIS → 카탈로그 등)

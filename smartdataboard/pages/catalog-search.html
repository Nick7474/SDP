운영자 온보딩

AI 데이터 카탈로그 사용 가이드

검색부터 API 승인까지

운영자가 자주 수행하는 카탈로그 업무를 6단계로 따라갈 수 있도록 정리했습니다.

1

데이터 검색

데이터셋명, 키워드, 부서, 파일 형식, 공개 여부 또는 자연어 질문으로 필요한 데이터를 찾습니다.

  • AI 질문으로 관련 데이터셋을 추천받습니다.
  • 결과에서 상세, 메타데이터, 다운로드, API 신청으로 이동합니다.
2

데이터셋 등록

신규 데이터는 기본 정보, 원천 시스템, 제공 방식, 갱신 주기, 담당 부서를 입력해 검토 상태로 등록합니다.

  • 수동 등록은 카탈로그 관리 화면에서 진행합니다.
  • 미등록 연계 데이터는 AI 메타데이터 추천 화면에서 등록 흐름을 시작합니다.
3

메타데이터 관리

설명, 담당 부서, 갱신 주기, 품질 점수, 공개 등급, 제공 형식의 누락 여부를 점검합니다.

  • 검색 결과의 메타데이터 드로어에서 DCAT-AP 항목을 확인합니다.
  • 편집은 AI 메타데이터 추천 화면에서 수행합니다.
4

표준용어 매핑

컬럼명과 표준용어 후보를 비교하고, AI 추천 유사도와 의미를 검토해 최종 매핑을 확정합니다.

  • 미매핑 컬럼은 검토 대기 상태로 관리합니다.
  • AI 추천은 자동 반영되지 않고 운영자 승인 후 확정됩니다.
5

AI 추천 검토

유사 데이터셋, 품질 개선 항목, 공개 가능성, API 제공 방식 추천을 확인하고 승인·보류·반려합니다.

  • 추천 결과는 업무 보조 정보이며 운영자 검토가 최종 기준입니다.
  • LLM 기반 자동 분석은 향후 고도화 범위로 확장할 수 있습니다.
6

API 승인 절차

API 신청 내역의 공개 등급, 보안 검토, 사용 목적을 확인하고 승인 또는 반려합니다.

  • 승인 이력과 제공 상태는 데이터 접근 관리 화면에서 확인합니다.
  • 제한공개 데이터는 별도 권한 검토 후 제공됩니다.

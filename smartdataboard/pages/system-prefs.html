언제 데이터 수집 실패, 연계 API 오류, 센서 미응답, 임계치 초과가 발생하면 알림이 생성됩니다.

어디서 상단 알림 벨에서 1차 확인하고, 운영 모니터링의 상태 모니터링 알림 목록에서 상세 확인합니다.

무엇을 발생 조건, 영향 범위, 심각도, 기본 담당 부서, SLA를 확인합니다.

어떻게 알림 상세 드로어에서 담당자를 지정하고 원인·조치 결과를 등록한 뒤 완료 처리합니다.

왜 알림을 단순 통지가 아니라 책임자 지정과 조치 이력까지 남는 운영 업무로 관리하기 위해서입니다.
